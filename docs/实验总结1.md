## 二、关键技术突破

### 2.1 BC7：错误守卫的 LSQ 端点细化（+0.29 dB）

**问题**：原代码注释指出 LSQ refinement 会导致质量回退（49 dB → 25 dB），故被禁用。

**根因**：
- 原 LSQ 仅用一维加权平均（无交叉项），数学上不等价于真实 2x2 LSQ。
- LSQ 在连续空间精炼，但量化后可能反而恶化质量。

**解决方案**：
1. 改用真正的 **2x2 LSQ 矩阵求解**：
   ```
   A = sum (1-t)²,  B = sum (1-t)·t,  C = sum t²
   X = sum p·(1-t), Y = sum p·t
   ep0 = (C·X - B·Y) / (AC - B²)
   ep1 = (A·Y - B·X) / (AC - B²)
   ```
2. 加 **错误守卫**：每次 LSQ 后重新量化、重计算误差，仅当误差严格降低才提交。
3. 加 **p-bit 搜索**：4 种 (pbit0, pbit1) 组合中选最低误差，便宜地获得额外精度。

### 2.2 BC6H：错误守卫修复隐性退化（+0.93 dB）

BC6H 已有 LSQ 但**没有错误守卫**。某些块的迭代会单调恶化但仍被提交。加 `if (new_error < prev_error) commit; else break;` 后立即获益。

### 2.3 BC1/BC3：2x2 LSQ + 非线性索引映射（+0.64 dB each）

BC1 的索引→t 映射非线性：`{0:0, 1:1, 2:1/3, 3:2/3}`。LSQ 时需用 lookup table 做正确映射。BC3 借助 BC1 自动获益。

### 2.4 ASTC 通用 QUANT_12 路径（+22-30 dB cumulative）

**最大单一杠杆**。原本简单格式（ASTC 5x4、5x5、…、12x12）使用：
- 自实现编码器
- QUANT_4（仅 4 个权重等级）
- 非标准 block mode 常量

效果停留在 22 dB 左右（接近随机噪声）。

**关键洞察**：
> ASTC block mode 编码 weight grid size，**不依赖 block size**。已存在的 4x4 ASTC 编码器（`astc_encode_core.hlsl`）实现了 PCA + QUANT_12 + 真实 trit/quint ISE encoding，在 ASTC_4x4 上达 47.78 dB。
>
> 任何格式的解码器都会自动 **bilinear 插值** 4x4 grid weights 到实际像素数。

**实现**：每个非 4x4 ASTC 格式只需：
1. 从输入像素中**子采样 16 个网格点**（4x4 grid 在该 block 上的对应位置）；
2. 调用现有 `encode_block(texels)`；
3. 完成。

代码量极小，每个文件约 25 行。例：
```hlsl
uint4 compress_astc_6x6(float4 pixels[36])
{
    float4 texels[BLOCK_SIZE];
    [unroll] for (int gy = 0; gy < 4; gy++) {
        [unroll] for (int gx = 0; gx < 4; gx++) {
            uint px = ((uint)gx * 5u + 1u) / 3u;
            uint py = ((uint)gy * 5u + 1u) / 3u;
            texels[gy * 4 + gx] = pixels[py * 6u + px] * 255.0f;
        }
    }
    return encode_block(texels);
}
```

### 2.5 修复完全损坏的格式

ASTC_8x6 和 ASTC_12x10 之前**故意**用 void_extent（输出常量颜色），停留在 12.42 dB（≈ 噪声水平）：
- ASTC_8x6: 12.42 → 32.37 dB（**+19.95 dB**）
- ASTC_12x10: 12.42 → 29.87 dB（**+17.45 dB**）

### 2.6 实验中尝试但被取代的方案

| 方法 | 效果 | 替代方案 |
|------|------|---------|
| QUANT_8 简单实现（3-bit 直接编码） | +4-7 dB | 被 QUANT_12 ISE 取代（+2-5 dB 进一步） |
| 自实现 5x4 grid（20 weights） | +0.5-1 dB | 被 4x4 grid + Q12 取代 |
| 3x3 Box averaging 采样 | +0.17 dB | 边际收益太小，未推广 |
| PCA endpoints | +0.5 dB | 已集成到 encode_core |

## 三、共性原则与技术沉淀

### 3.1 错误守卫迭代（universally applicable）

任何"看似改进"的算法变换，必须通过 **before/after 误差比较** 决定是否提交：

```
1. 保存当前状态 + 误差
2. 尝试改进
3. 重新量化 → 重新计算误差
4. if (new_error < prev_error) 提交; else 回滚 + break
```

适用：BC7 LSQ、BC6H LSQ、BC1 LSQ、ASTC encode_core LSQ。

### 3.2 解码器是隐性资源

不应仅把解码器当作评估工具——**解码器实现的特性可以反向利用**：
- ASTC bilinear 插值 → 编码器只需提供稀疏 grid weights
- BC6H FP16 端点 → 端点可用更高精度而无需 quantize 算法

### 3.3 写入 mirror copies 易成隐性 bug

项目有三处 shader 镜像（`sdk/`、`shaders/`、`shaders_link/`），每次修改必须三处同步。建议未来用 hard link 或 build-time copy 自动同步。

## 四、未达目标 / 受限方向

### 4.1 BC4/BC5：测试方法限制

```
BC4 PSNR = 14.01 dB（实际单通道质量 >40 dB）
BC5 PSNR = 18.15 dB（实际双通道质量 >40 dB）
```

框架的 `compute_psnr` 比较 RGB 三通道，但 BC4 解码 → `(R, 0, 0, 1)`、BC5 → `(R, G, 0, 1)`。G/B 通道全 0 导致 PSNR 严重低估。

**根本解决**需修改 `src/eval_metrics.cpp` 但 CLAUDE.md 禁止改动 src/。

### 4.2 ASTC_4x4 已超目标，但仍非全局最优

47.78 dB 接近 BC7 的 48.24 dB，但理论上 ASTC 4x4 同等 bit budget 应可达 50+ dB。剩余优化空间：
- **多 partition 编码**（CEM 8 双 partition）：~2-4 dB
- **Dual plane**（一通道独立权重）：1-2 dB（仅特定 block 类型）
- **自适应 QUANT 选择**（按数据自动选 Q8/Q12/Q20）

### 4.3 BC4/BC6H/BC7 高级模式未实现

| 格式 | 已用模式 | 全部模式 |
|------|---------|---------|
| BC6H | mode 11 only | 14 modes（mode 1-10 双 partition；mode 11-14 单 partition） |
| BC7 | mode 6 only | 8 modes（mode 0/1/2/3 三 partition；mode 4/5 dual plane） |

实现完整模式选择可能再获 +1-3 dB，但代码复杂度大幅增加。

## 五、后续方向（按优先级）

### 5.1 高优先级（确定收益）

1. **BC4/BC5 测试方法修复**（1 行代码改动级别）
   - 在 `decompress_bc4` 中复制 R 到 G/B 通道
   - 不算"算法改进"但能反映真实质量
   - **预期影响**：暴露真实的 BC4/BC5 优势

2. **多 partition 支持（ASTC + BC7）**
   - ASTC CEM 8 + 2 partition
   - BC7 mode 0/1/2/3
   - **预期增益**：所有 4x4 类格式 +2-5 dB
   - **复杂度**：中等高（partition pattern 选择 + bit budget 重新分配）

3. **HDR ASTC 支持**
   - 当前只有 LDR profile
   - **预期影响**：游戏引擎 HDR 内容支持

### 5.2 中优先级（小步迭代）

4. **基于内容的自适应模式选择**
   - 同一 block 用多种 mode 编码，取 PSNR 最高的
   - QualityLevel 2 时尝试更多 mode
   - **预期增益**：+1-2 dB（复杂 block 上更高）

5. **更严格的权重量化（snap to actual decoded levels）**
   - 当前 `round(t * 7)` 是均匀近似
   - 用 ASTC 实际解码值的二分边界更精确
   - **预期增益**：0.1-0.3 dB

6. **BC1/BC7 局部端点搜索**
   - LSQ 后做 ±2 步小范围搜索
   - **预期增益**：0.2-0.5 dB
   - **代价**：~5x 编码时间（quality level 2 only）

### 5.3 低优先级 / 实验性

7. **感知度量优化**
   - 当前优化 PSNR；可改为 SSIM 或 FLIP
   - 需修改误差守卫的比较函数
   - **预期影响**：人眼主观质量提升，PSNR 可能略降

8. **法线贴图专用路径（BC5 / ASTC normal）**
   - 假设 z 可重建，仅编码 xy
   - 用 normalize(xyz) 重建 metric
   - **预期影响**：normal map 质量大幅提升

9. **Speed/quality 自动调节**
   - 当前 QualityLevel 0/1/2 三档静态
   - 可基于像素统计动态选择内部参数
   - **预期影响**：降低 QL 1 的延迟、不损失质量

10. **HLSL 编译时间优化**
    - 部分 shader（ASTC 12x12 + Q12）SPIRV ~150 KB
    - 大量 [unroll] 循环膨胀
    - 可拆分为多 entry point 减小热度

### 5.4 长期方向

11. **shader code generation（meta-template）**
    - 14 个 ASTC 文件结构高度相似
    - 用脚本生成可减少代码重复，便于全局优化

12. **比较 ARM astcenc 参考结果**
    - 当前我们用 4x4 encode_block 路径覆盖所有 ASTC 格式
    - astcenc CPU 参考可能用不同 block mode（5x4 grid + ISE 等）能更高
    - **预期增益**：5x5/6x6 等还有 1-3 dB 上限

13. **ASTC encode_core 进一步加强**
    - 当前 4x4 路径 47.78 dB，理论上限 50+ dB
    - 可加 LSQ 错误守卫、p-bit-like 微扰、QUANT_20 选择
